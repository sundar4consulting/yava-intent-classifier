What is a Skill?
A Skill is a registered service in Watson Orchestrate that exposes one or more Tools via REST API. Skills are defined using OpenAPI 3.0.3 specifications and can be discovered and invoked by Agents.

Backend Elasticsearch + RAG + OpenAI GPT-4o

Tool 1: classify (Primary Tool)

Purpose

Complete intent classification with all NLU features bundled: classification, slot extraction, multi-intent detection, disambiguation, and LLM fallback.

Tool Signature

Property Value
operationId classifyIntent
Method POST
Endpoint /v2/classify

Input Parameters

{ "user_input": "string (required)", // User's utterance "conversation_id": "string (optional)", // Session ID for context "member_id": "string (optional)", // Member identifier "context_aware": "boolean (default: true)", // Use conversation history "force_llm": "boolean (default: false)" // Force LLM invocation }

Output Schema

{ // Core Classification "intent": "string", // Detected intent name "agent": "string", // Target agent (e.g., "PharmacyAgent") "category": "string", // Intent category "confidence": "float (0-1)", // Classification confidence // V2 Metadata "classification_method": "RAG-only | RAG+LLM", "llm_invoked": "boolean", "vector_store": "elasticsearch", // LLM Enhancement (if invoked) "llm_reasoning": "string", // Natural language explanation "llm_confidence": "float", "needs_clarification": "boolean", // Entity Extraction "slots": {}, // Extracted entities // Multi-Intent Detection "has_multi_intents": "boolean", "multi_intents": [ { "segment": "string", "intent": "string", "confidence": "float" } ], // Performance Metrics "performance": { "rag_time_ms": "float", "llm_time_ms": "float", "llm_cost_usd": "float" } }

Agent Definition File: agent_v3_llm_aware_fallback.yaml
This YAML file defines the agent's behavior, LLM model, available tools, and decision-making logic.

Agent Specifications

Property Value Description
Name YAVA_Intent_Classifier_V3_RAG_LLM_Fallback Agent identifier
LLM Model groq/openai/gpt-oss-120b Large language model
Tools Available classifyIntent, listIntents 2 tools from skill
Style default Conversation style
Spec Version v1 Watson Orchestrate format

Agent Instructions (Key Features)

1. LLM Availability Detection

Agent checks if LLM was invoked by the classifier tool:

if (llm_invoked == false AND confidence < 0.75): # LLM unavailable - Agent applies manual reasoning

2. Manual Reasoning Protocol

When backend LLM is unavailable, agent provides reasoning enhancement:

  • Analyze utterance for healthcare keywords
  • Check conversation context
  • Apply domain-specific rules
  • Boost confidence if patterns are clear
  • Generate clarification prompts if needed

3. Response Strategy by Confidence

Confidence Range LLM Available LLM Unavailable
HIGH (â‰¥ 0.75) Report confidently Report confidently
MEDIUM (0.60-0.74) Trust LLM reasoning Apply manual reasoning
LOW (< 0.60) Ask clarifying questions Ask clarifying questions
